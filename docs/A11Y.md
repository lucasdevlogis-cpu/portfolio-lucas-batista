# Acessibilidade

> Guia canônico. Meta: WCAG 2.1 AA e Lighthouse a11y 100. Estado local em 20/07/2026: **100 desktop e 100 mobile**.

## Regras obrigatórias

| Área | Regra |
|---|---|
| Contraste | texto normal 4,5:1; texto grande e componentes 3:1 |
| Teclado | ordem lógica, foco visível, ESC fecha modal, retorno ao acionador |
| Nome acessível | o texto visível deve fazer parte do nome anunciado |
| Touch | alvos de pelo menos 44×44 px |
| Movimento | respeitar `prefers-reduced-motion`; conteúdo essencial não depende de animação |
| Estrutura | `header`, `nav`, `main`, `section`, `article`, `footer`; um H1 por página |
| Imagens | `alt` útil; ícones com texto usam `aria-hidden` |
| Dados visuais | gráfico recebe resumo textual; mapa é região nomeada e declara limitações |

## Tokens de contraste

| Uso | Token |
|---|---|
| Texto forte em claro | `--ink` / `--primary` |
| Texto secundário em claro | `--muted-foreground` |
| Teal como texto ou fundo com branco | `--accent-contrast` |
| Dourado como texto em claro | `--warm-accent-contrast` |
| Texto secundário em escuro | `--text-on-dark-muted` |
| Acento em escuro | `--text-on-dark-accent` |

`--accent` e `--warm-accent` são sinais visuais; não usar como texto pequeno em fundo claro. A fonte editável é [`design/tokens.json`](../design/tokens.json).

## Implementação atual

- Skip link para `#conteudo`.
- `Dialog` gerencia trap de foco, ESC e retorno ao botão.
- Filtros da biblioteca anunciam contagem via `aria-live="polite"`.
- CTAs de demo usam nomes como `Explorar Frete: Simulador de Custo de Frete`, preservando o rótulo visível.
- Cards de contato dependem do texto visível; `aria-label` não substitui conteúdo útil.
- ECharts recebe `role="img"` e resumo de categorias/valores.
- MapLibre recebe `role="region"`, nome descritivo, controles e atribuição visível.
- Analytics só é montado na Vercel, evitando erro 404 local.
- O conteúdo LCP do hero é estático; reveals abaixo da dobra seguem o `MotionProvider`.

## Checklist manual

1. Navegar sem mouse pela homepage e por cada rota `/provas/{slug}`.
2. Abrir e fechar modal com teclado; confirmar foco no acionador.
3. Operar filtros e ouvir o anúncio da nova contagem.
4. Ativar `prefers-reduced-motion` e confirmar ausência de movimento não essencial.
5. Testar em NVDA: headings, links de contato, resumo dos gráficos e região do mapa.
6. Verificar zoom de 200% e ausência de overflow horizontal em 375 px.

## Automação

```bash
npm run test:e2e
npm run lighthouse:all
```

O Lighthouse é gate, não substitui NVDA nem inspeção manual. Uma regressão deve ser corrigida antes do commit; não registrar 100 sem nova medição.
