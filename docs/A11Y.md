## 1. Checklist WCAG 2.1 AA

### 1.1 Contraste

| Elemento | Ratio Mínimo | Como Testar |
|----------|:------------:|-------------|
| Texto normal (< 18px) | 4.5:1 | axe DevTools ou WebAIM Contrast Checker |
| Texto grande (>= 18px bold / >= 24px) | 3:1 | axe DevTools no Chrome |
| Componentes interativos (botoes, inputs) | 3:1 | Comparar cor de fundo vs cor de borda/fundo adjacente |

**Tokens do design system já verificados:**

| Token | Hex | Fundo | Ratio | Status |
|-------|-----|-------|-------|--------|
| Ink | `#07111f` | Editorial `#f5f2ed` | 15.8:1 | ✅ |
| Muted foreground | `#556070` | Editorial `#f5f2ed` | 4.8:1 | ✅ |
| Accent | `#16a99c` | Surface dark `#07111f` | 4.9:1 | ✅ |
| Warm accent contrast | `#7a5a1a` | Card / Editorial | ≥ 4.5:1 | ✅ texto pequeno |
| Warm accent | `#c9983f` | Surface dark `#07111f` | 8.2:1 | ✅ decorativo / dark |
| Border | `#c8c2b8` | Editorial `#f5f2ed` | 1.6:1 | ⚠️ Apenas divisórias, nunca texto |

- [ ] Placeholders em inputs (se houver)
- [ ] Texto pequeno em badges e labels
- [ ] Elementos focáveis dentro de `aria-hidden`

### 1.4 Animações

`components/providers/MotionProvider.tsx` já implementa `useReducedMotion()` via `MotionConfig`:

```tsx
"use client";

import { LazyMotion, MotionConfig, domAnimation, useReducedMotion } from "framer-motion";

function ReducedMotionConfig({ children }: { children: React.ReactNode }) {
  const prefersReducedMotion = useReducedMotion();
  return (
    <MotionConfig reducedMotion={prefersReducedMotion ? "always" : "never"}>
      {children}
    </MotionConfig>
  );
}

export function MotionProvider({ children }: { children: React.ReactNode }) {
  return (
    <LazyMotion features={domAnimation}>
      <ReducedMotionConfig>{children}</ReducedMotionConfig>
    </LazyMotion>
  );
}
```

- [x] `LazyMotion` + `domAnimation` economiza bundle
- [x] `viewport={{ once: true }}` evita re-animação
- [x] Estados iniciais com `opacity: 1` (conteúdo nunca invisível)
- [x] **`useReducedMotion()`:** Respeitado em `MotionProvider`

## 3. Testes de Acessibilidade

### Teste Manual (5 minutos)
1. Desconectar mouse
2. Navegar com `Tab` por toda a página
3. Verificar se foco é visível em todos os elementos interativos
4. Ativar cada elemento com `Enter` ou `Space`
5. Fechar modal com `Escape`
6. Verificar ordem de foco lógica

### Teste com Leitor de Tela (10 minutos)
1. Ativar NVDA (Windows) ou VoiceOver (Mac)
2. Navegar com `Tab` e setas
3. Verificar se todos os elementos são anunciados
4. Verificar se a ordem faz sentido
5. Testar filtros de cases (devem anunciar resultado)

### Ferramentas Automatizadas
```bash
# axe DevTools (extensão Chrome)
# Instalar e rodar scan em cada seção

# Lighthouse CI
npm install -D @lhci/cli
# Meta: a11y >= 100

# pa11y (CLI)
npm install -g pa11y
pa11y https://portfolio-lucas-batista-murex.vercel.app --standard WCAG2AA
```

## 4. Meta: Lighthouse A11Y 100

Para subir de 96 para 100, verificar especificamente:

| Possível Problema | Pontos | Solução |
|-------------------|--------|---------|
| `[aria-hidden="true"]` contém elementos focáveis | 2-4 | Remover foco de elementos dentro de aria-hidden |
| Contraste insuficiente em texto pequeno | 2-4 | Ajustar cor ou tamanho |
| IDs duplicados | 1-2 | Garantir IDs únicos |
| Ordem de foco incorreta | 1-2 | Verificar DOM order vs visual order |
| Links sem texto discernível | 1-2 | Adicionar aria-label ou texto visível |
