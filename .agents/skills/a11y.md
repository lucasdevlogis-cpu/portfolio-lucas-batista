Regras de acessibilidade obrigatórias para toda a landing. Meta: Lighthouse a11y **100**.

## Regras Obrigatorias

### Contraste
- Texto normal (< 18px): ratio mínimo **4.5:1**
- Texto grande (>= 18px bold): ratio mínimo **3:1**
- Componentes interativos: ratio mínimo **3:1**
- Usar tokens do design system já verificados (ver `docs/A11Y.md` seção 1.1)
- Nunca usar texto sobre imagens sem overlay escuro

### Teclado
- **Skip link:** já em `layout.tsx` (`#conteudo`)
- **Ordem de foco lógica:** Header → Hero CTAs → EvidenceStrip → Perfil → Cases → Trajetória → Contato → Footer
- **Focus visible:** usar classe `.focus-ring` (tokens em `globals.css`)
- **Escape fecha modais:** Dialog do shadcn já gerencia
- **Tab trap em modais:** Dialog do shadcn já gerencia
- **Ancoras de navegação:** funcionam com Enter

### Screen Readers
- Ícones: `aria-hidden="true"` quando com texto, `aria-label` quando sozinhos
- ProfileCockpit: `role="img"` + `aria-label` ou `aria-hidden="true"` se decorativo
- Filtros: anunciar resultado via `aria-live="polite"`
- Modal: `aria-modal="true"`, título em `DialogTitle`, descrição em `DialogDescription`
- Timeline: estrutura semântica com `<article>` para experiências
- Evidence strip: usar `<section aria-label="...">`

### Framer Motion
- `useReducedMotion()` deve ser respeitado no `MotionProvider`
- Estado inicial `opacity: 1` (conteúdo nunca invisível)
- `viewport={{ once: true }}` em animações de scroll

### Semantica
- `<main id="conteudo">` para conteúdo principal ✅
- `<section id="perfil|cases|trajetoria|contato">` para cada seção ✅
- `<article>` para cards de case ✅
- `<nav>` para navegação ✅
- Hierarquia H1 > H2 > H3 sem pulos
- H1 único por página

## Anti-Padrões (Nunca Faca)
- `onClick` em `<div>` sem `role="button"` e `tabIndex={0}`
- Ícones sem `aria-hidden` ou `aria-label`
- Cores hex inline (sempre usar tokens)
- Animações que escondem conteúdo essencial
- `outline: none` sem substituir por focus visível

```tsx

```

### EvidenceStrip
```tsx
<section aria-label="Métricas principais do portfólio">
  {metrics.map(...)}
</section>
```

### CaseLibrary (filtros)
```tsx

```
